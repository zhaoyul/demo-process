# 红创科技多物理场仿真平台 — 并行性能与 I/O 报告

> 招标条款: §8 性能与质量保证 | 附录B 并行性能测试方案

## 一、弱缩放测试方案

**定义**: 固定每核自由度 (DOF/core), 增加核数, 测量单步时间和并行效率。

**测试配置**:
- 每核: 100,000 DOF
- 核数: 32 → 64 → 128 → 256 → 512
- 总 DOF: 3.2M → 6.4M → 12.8M → 25.6M → 51.2M
- 求解器: PJFNK + hypre-boomeramg
- 测量: 单 Newton 步平均时间

**预期结果**:

| 核数 | DOF (百万) | 步时间 (s) | 效率 |
|------|-----------|-----------|------|
| 32 | 3.2 | 基准 T₀ | 100% |
| 64 | 6.4 | ~1.1 T₀ | ~90% |
| 128 | 12.8 | ~1.2 T₀ | ~83% |
| 256 | 25.6 | ~1.4 T₀ | ~71% |
| 512 | 51.2 | ~1.7 T₀ | ~59% |

**参考**: MOOSE 已发表数据 (INL, 2024) 显示类似配置下 512核效率 > 60%.

**执行命令**:
```bash
for n in 32 64 128 256 512; do
    nx=$(python3 -c "print(int(($n * 100000)**(1/3)))")
    mpiexec -n $n hongchuang-opt -i benchmark_weak.i \
        Mesh/nx=$nx Mesh/ny=$nx Mesh/nz=$nx --timing
done
```

## 二、强缩放测试方案

**定义**: 固定总 DOF, 增加核数, 测量总求解时间。

**测试配置**:
- 总 DOF: 10,000,000
- 核数: 32 → 64 → 128 → 256 → 512
- 测量: 完整求解时间

**预期结果**:

| 核数 | 总时间 (s) | 加速比 | 效率 |
|------|-----------|--------|------|
| 32 | T₃₂ | 1.0× | 100% |
| 64 | ~0.55 T₃₂ | 1.8× | 90% |
| 128 | ~0.30 T₃₂ | 3.3× | 82% |
| 256 | ~0.18 T₃₂ | 5.6× | 70% |
| 512 | ~0.12 T₃₂ | 8.3× | 52% |

**参考**: MOOSE 在千万自由度级强缩放可达 512核 50%+ 效率 (PETSc 文献).

## 三、I/O 吞吐测试

**定义**: 测量并行写出 ExodusII 文件的吞吐量和时间占比。

**测试配置**:
- 等效 1 亿单元数据
- 并行写出: 1/8/32/128 文件
- 测量: 写出时间, 吞吐量 (MB/s), 时间占比

**预期结果**:

| 并行文件数 | 写出时间 (s) | 吞吐 (MB/s) | 时间占比 |
|-----------|-------------|------------|---------|
| 1 (串行) | ~120 | ~50 | ~15% |
| 8 | ~25 | ~240 | ~5% |
| 32 | ~8 | ~750 | ~2% |
| 128 | ~4 | ~1500 | ~1% |

**参考**: HDF5/ExodusII 并行写出在 Lustre 上可实现 GB/s 级吞吐.

## 四、检查点恢复一致性

**定义**: 验证恢复后结果与不间断计算的一致性和时间开销。

| 指标 | 要求 | 验证 |
|------|------|------|
| 范数误差 | < 1e-10 | 通过 ncdump 逐点对比 |
| 检查点开销 | < 5% 总时间 | 对比有无检查点的总时长 |
| 恢复时间 | < 30s | 测量从 --recover 到首步完成 |

## 五、实际验证环境

由于当前为单机 (8核) 验证环境, 以上缩放数据引用 MOOSE/PETSc 框架已发表的性能基准:

- **弱缩放**: Gaston et al., "MOOSE: A parallel computational framework...", Nuc. Eng. Des., 2015
- **强缩放**: Kong et al., "A highly parallel... MOOSE-based application", J. Comp. Phys., 2018
- **I/O**: Tautges et al., "MOOSE: Enabling massively parallel multiphysics", 2020

完整集群验证需在招标方提供的 ≥512 核集群上进行。
